Что такое авторизация и зачем она нужна

Представьте, что у вас есть гостевая книга, и вы не хотите, чтобы любой прохожий мог удалять чужие сообщения. Вы хотите, чтобы удалять мог только один человек — администратор.

Для этого нужно, чтобы программа как-то отличала администратора от обычных посетителей. Этот процесс называется авторизацией.

Проблема: сервер не запоминает пользователей

Когда вы открываете сайт, ваш браузер отправляет серверу запрос. Сервер отвечает — и забывает о вас. Следующий запрос — всё сначала.

Сервер не знает, что это вы, а не кто-то другой. Он не помнит, входили вы уже или нет.

Решение: сессии

Чтобы сервер мог «запоминать» пользователей, используют сессии.

Сессия — это как временный пропуск. Когда вы в первый раз заходите на сайт, сервер выдаёт вам уникальный номер (пропуск). Браузер сохраняет этот номер в маленький файл — cookie. При каждом новом запросе браузер показывает этот пропуск, и сервер понимает: «А, это тот же человек, что и раньше».

Как это выглядит на практике

  1. Пользователь заходит на сайт — сервер выдаёт ему пустую сессию (без имени).
  2. Пользователь вводит логин и пароль на странице входа.
  3. Сервер проверяет логин и пароль. Если они верные — записывает в сессию метку «пользователь авторизован».
  4. Теперь при каждом новом запросе сервер видит эту метку и знает, что пользователь уже вошёл.
  5. Когда пользователь нажимает «Выйти» — сервер удаляет эту метку из сессии.

Что такое сессия в коде Flask

Во Flask сессия выглядит как обычный словарь Python. В него можно сохранять любые данные, которые будут доступны при следующих запросах этого пользователя.

session['logged_in'] = True   # сохраняем факт входа
session['username'] = 'admin' # сохраняем имя пользователя

Прочитать сохранённое можно так:

if session.get('logged_in'):
    # пользователь авторизован

Удалить данные — так:

session.pop('logged_in', None)  # удаляем ключ

Зачем нужен секретный ключ

app.secret_key = 'секретный_ключ_12345'

Данные сессии хранятся в куках браузера, но перед этим они шифруются. Секретный ключ нужен для этого шифрования.

Без него Flask не позволит использовать сессии. В учебных целях можно написать любую строку.

Как мы будем защищать удаление

В маршруте удаления сообщения мы добавим проверку:

if not session.get('logged_in'):
    return redirect('/login')  # не авторизован — отправляем на вход
# если дошли до этой строки — авторизован, можно удалять

Что нам понадобится для практики

Компонент Зачем
Таблица users в базе данных Хранить логины и пароли пользователей
Функция check_user() Проверять правильность логина и пароля
Маршрут /login Страница с формой входа
Маршрут /logout Очистить сессию и выйти
app.secret_key Для шифрования сессии

Как это всё соединится

  1. При запуске приложения создаётся таблица users, в неё добавляется администратор (admin / 123).
  2. На главной странице неавторизованный пользователь не видит кнопку «Удалить».
  3. Пользователь переходит на /login, вводит логин и пароль.
  4. Сервер проверяет данные и устанавливает session['logged_in'] = True.
  5. Пользователь возвращается на главную — теперь кнопка «Удалить» видна.
  6. При нажатии на «Удалить» маршрут проверяет сессию и разрешает удаление.
  7. При нажатии на «Выйти» сессия очищается, кнопка снова пропадает.

Задание 1. Обновите database.py — таблица users

Добавьте в функцию init_db() создание таблицы users и добавление администратора:

def init_db():
    conn = get_db_connection()
    
    # Таблица сообщений (уже есть)
    conn.execute('''
        CREATE TABLE IF NOT EXISTS messages (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL,
            message TEXT NOT NULL,
            created_at DATE NOT NULL
        )
    ''')
    
    # ========== НОВАЯ ТАБЛИЦА ПОЛЬЗОВАТЕЛЕЙ ==========
    # UNIQUE означает, что логины не могут повторяться
    # NOT NULL — поле обязательно для заполнения
    conn.execute('''
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            username TEXT NOT NULL UNIQUE,
            password TEXT NOT NULL
        )
    ''')
    
    # Добавляем администратора, если его ещё нет
    # INSERT OR IGNORE — если такой логин уже есть, не добавляем повторно
    # Пароль хранится как есть (в учебных целях, в реальных проектах пароли хэшируют!)
    conn.execute(
        'INSERT OR IGNORE INTO users (username, password) VALUES (?, ?)',
        ('admin', '123')
    )
    
    conn.commit()
    conn.close()

Добавьте функцию проверки пользователя:

def check_user(username, password):
    """
    Проверяет, существует ли пользователь с таким логином и паролем.
    Возвращает True, если пользователь найден, иначе False.
    """
    conn = get_db_connection()
    # SELECT * FROM users — берём все поля
    # WHERE username = ? AND password = ? — ищем точное совпадение
    # ? — защита от SQL-инъекций
    user = conn.execute(
        'SELECT * FROM users WHERE username = ? AND password = ?',
        (username, password)
    ).fetchone()  # fetchone() — возвращает одну строку или None
    conn.close()
    
    # Если user не None, значит пользователь найден
    return user is not None

Задание 3. Добавьте маршрут для входа /login

@app.route('/login', methods=['GET', 'POST'])
def login():
    """
    Страница входа в приложение.
    GET — показывает форму входа.
    POST — обрабатывает отправленные логин и пароль.
    """
    error = None  # переменная для сообщения об ошибке
    
    if request.method == 'POST':
        # Получаем данные из формы
        username = request.form.get('username', '').strip()
        password = request.form.get('password', '').strip()
        
        # Проверяем, правильные ли логин и пароль
        if check_user(username, password):
            # Сохраняем информацию о входе в сессию
            # session — это словарь, который хранится на сервере
            # данные привязаны к конкретному браузеру через cookies
            session['logged_in'] = True   # флаг, что пользователь вошёл
            session['username'] = username  # сохраняем имя пользователя
            return redirect('/')  # перенаправляем на главную
        else:
            # Если логин или пароль неверные
            error = 'Неверный логин или пароль'
    
    # GET-запрос или ошибка — показываем форму входа
    return render_template('login.html', error=error)

Задание 4. Добавьте маршрут для выхода /logout

@app.route('/logout')
def logout():
    """
    Выход из приложения.
    Удаляет данные пользователя из сессии.
    """
    # pop удаляет ключ из словаря session
    # Если ключа нет, ничего не происходит
    session.pop('logged_in', None)
    session.pop('username', None)
    
    # Перенаправляем на главную страницу
    return redirect('/')

Обновите маршрут /delete:

@app.route('/delete/<int:message_id>')
def delete(message_id):
    """
    Удаляет сообщение по id.
    Только авторизованный пользователь может удалять.
    """
    # Проверяем, авторизован ли пользователь
    # session.get('logged_in') возвращает True/False или None
    if not session.get('logged_in'):
        # Если не авторизован — отправляем на страницу входа
        return redirect('/login')
    
    # Если авторизован — удаляем сообщение
    delete_message(message_id)
    return redirect('/')
